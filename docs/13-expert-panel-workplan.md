## 핵심 발견 (5줄 요약)

1. **쿼터 수학이 안 맞는다** — 최악 63콜/일 vs 한도 20. 배칭+예산 없이는 모든 후속 기능이 굶는다. (×3: AI·아키텍트·PM)
2. **파이프라인이 크래시에 약하다** — 당일 재실행 시 데일리 덮어쓰기 유실, 깨진 jsonl 한 줄이 cron 영구 사망, 중간 크래시 시 요약 콜 증발. (×3: 코드·QA·아키텍트)

4. **학습 카드에 렌더 버그** — 코드펜스 안 `[[ ]]`가 위키링크로 오변환 (코딩 채널이라 재현 확률 높음). (코드)
5. **eval 부재** — 테스트 79개 전부 FakeClient라 프롬프트 품질 회귀를 하나도 못 잡음. 골든셋 12건 설계 나옴. (AI)

## 전문가 충돌 판정 (PM)

- **'오늘' 동선**: 웹UX는 nav 링크(P1·S), PM제품은 첫화면 개편(P2·M) → **nav 링크 먼저, 개편은 P2**. 싼 것부터.
- **eval 시점**: AI전문가 P1 vs 쿼터 소진 현실 → **쿼터 예산(#1) 뒤로**. eval 자체가 콜을 쓰므로 예산 없이 돌리면 본업(요약)을 굶김.
- **적용형 학습 노트 설계**: AI전문가의 "grounded/generative 섹션 분리 + profile.yaml 주입 + 하루 1콜 배칭"을 스펙(#12)에 채택.

## Batch 1 실행 프롬프트 카드 (서브에이전트 위임용)

> 각 카드는 독립 위임 단위. 공통 규칙: TDD(테스트 먼저), 기존 79개 테스트 회귀 금지,
> 기존 주입 인터페이스(client/clients 파라미터) 유지, 커밋은 카드당 1개.

**카드 A — LLM 게이트웨이 + 쿼터 예산** `(#3→#1→#2)` 역할: 백엔드 코더
> collector/summarize.py의 complete_text·_api_keys·_collect_keys·_is_quota_error를 collector/llm.py로 추출하라(기존 import 경로는 재수출로 호환 유지). llm.py에 (a) state/llm_budget.json 일일 콜 카운터 — 날짜 바뀌면 리셋, 18콜 도달 시 QuotaExhausted 발생, (b) 전 키 429 시 같은 실행 내 이후 호출 즉시 스킵(서킷브레이커), (c) 503/overloaded 감지 시 짧은 backoff 후 같은 키 1~2회 재시도(429 로테이션과 분리)를 구현하라. pipeline·wiki는 QuotaExhausted를 기존 [retry-later] 경로로 처리. 완료 조건: 신규 테스트(카운터 리셋·차단·503 재시도·서킷브레이커) + 전체 suite 통과.

**카드 B — 배치 요약** `(#1 후속)` 역할: 백엔드 코더 + AI전문가
> summarize.py에 여러 Item을 한 콜로 요약하는 batch_summarize를 추가하라: 뉴스형 4건/콜, 학습형 2건/콜. 프롬프트는 번호 매긴 항목 → 번호별 결과(요약+카테고리) JSON 배열로 받고, 항목별 파싱 실패 시 그 항목만 개별 콜 폴백. pipeline.run이 소스 순회 후 항목을 모아 배치로 부르도록 수정. 완료 조건: FakeClient 테스트로 배치 파싱·부분 실패 폴백 검증, 콜 수 계산 테스트(11피드×5건 시나리오에서 ≤20콜).

**카드 C — 파이프라인 내구성** `(#4)` 역할: 백엔드 코더
> (a) digest.write_digest가 기존 당일 파일이 있으면 항목을 병합(제목 링크 기준 dedup)하도록 수정 — 당일 재실행 시 아침 수집분 유실 방지. (b) pipeline.run에서 항목 성공 직후 단건 append_items+state.save로 즉시 영속화. (c) store.load_items가 깨진 줄을 경고 후 skip하도록 수정, StateStore·TopicStore 생성자도 corrupt JSON 시 백업 후 빈 상태로 시작. 완료 조건: corrupt-line·당일 재실행 병합·중간 크래시 시뮬레이션 테스트 통과.

**카드 D — 보안 3종** `(#5)` 역할: 보안 코더
> (a) web/render.py render_markdown 반환 직전 nh3(pip 추가) 허용목록 sanitize. (b) web/app.py debug를 환경변수 FLASK_DEBUG=1일 때만 켜기. (c) collector digest·topics의 링크 렌더에 http/https 스킴 검증 + 제목 대괄호 이스케이프 헬퍼 공유. 완료 조건: `<script>` 요약이 이스케이프되는 테스트, javascript: 링크 차단 테스트.

**카드 E — 웹 렌더버그 + 즉효 3종** `(#6, #7)` 역할: 프론트 코더
> (a) _replace_wikilinks가 코드펜스(```)와 인라인 코드 구간을 건너뛰도록 수정. (b) 학습 카드(‘**핵심 개념**’ 포함 기사)에 article--learning 클래스 + '학습' 뱃지. (c) .doc에 overflow-wrap:anywhere 등 모바일 가로스크롤 방지 + 맨몸 URL 자동 링크화. (d) nav '오늘' 링크(최신 데일리) + 데일리 상세에 이전/다음. 완료 조건: 코드펜스 보존 테스트 + 뱃지 렌더 테스트 + 기존 웹 테스트 통과.

**카드 F — 파싱 견고화 + 테스트 격리** `(#8, #9, #10)` 역할: 코더 + QA
> (a) '카테고리:' 추출을 마지막 비어있지 않은 줄에서 `\**\s*카테고리\s*[:：]` 정규식으로 뽑는 공용 헬퍼로 교체(summarize·classify 공유), LEARNING_PROMPT의 자기모순 문구 삭제, classify_item의 미사용 known_topics 제거. (b) categories.yaml 실파일에 결합된 테스트를 categories= 주입/tmp_path로 격리. (c) run_wiki 실패경로 3종 테스트 추가 — 특히 빈 분류 결과 시 seen 미마킹으로 바꿔 항목 유실 방지. 완료 조건: 전체 suite 통과.

**카드 G — 문서·스펙 (PM 직접, 에이전트 불필요)** `(#11, #12)`
> 00-overview 현행화(pytest 79, 쿼터 20콜/일 명시, 로드맵 순서+Q&A 승격조건 "노트 30일치"), docs/14 적용형 학습 노트 스펙 작성 — 미결정 5건 확정: profile.yaml 위치·형식 / 신규성 기준 / eval 방식 / 일일 쿼터 할당 / 출력 위치.
