# 12. 개발·학습 콘텐츠 확장 (A+B) — 설계

> "코드나 배울 게 없다"는 진단에서 출발. 뉴스 요약 일변도인 위키에 **배울 수 있는**
> 개발·학습 콘텐츠를 넣고, 그 항목은 일반 요약이 아니라 **학습 카드**로 재가공한다.

- **작성:** 2026-07-01
- **범위:** A(학습형 출처 추가) + B(학습 카드 재가공). C(RAG 튜터)는 별도, 백로그 유지.
- **원칙:** 기존 collect→summarize→wiki 골격을 그대로 두고 **증분 확장**.

## 핵심 결정

### 1. 학습 소스 구분 = 소스 단위 `learning` 플래그
`sources.yaml`의 각 출처에 `learning: true`를 달 수 있다. 이 플래그가
`config.Source.learning` → `Item.learning`으로 전파되어, 요약 단계에서 어떤
프롬프트를 쓸지 결정한다. (항목별 자동 판별은 YAGNI — 나중에.)

### 2. 학습 카드는 **헤딩(`#`)을 쓰지 않는다** ⚠️
`_render_item`([topics.py](../collector/topics.py))이 `item.summary`를
`### [제목]` 바로 밑에 그대로 넣는다. 학습 카드가 `## 핵심 개념` 같은 헤딩을 쓰면
정리형 위키의 `## 테마` 구조와 웹 렌더러의 기사 분리(`_wrap_articles`)가 **둘 다 깨진다.**
따라서 학습 카드는 **굵은 라벨 + 불릿 + 코드펜스**만 사용한다:

```
**핵심 개념**
- 개념/원리 (3~5개, 각 한 줄)

**코드·명령**
```lang
(원문에 코드/명령이 있을 때만. 없으면 이 섹션 통째 생략)
```

**실습 포인트**
- 따라 할 단계·팁 (2~4개)

**한 줄 정리**
한 문장.
```

이러면 `### 기사` 밑에 안전하게 nested 되고, 웹에서는 `<strong>`/`<pre><code>`로
렌더돼 충돌이 없다.

### 3. 카테고리 "개발·학습" 신설
`categories.yaml`에 추가. 학습 항목이 이 카테고리로 모여 전용 페이지를 이룬다.

## LEARNING_PROMPT (구성)

```
다음은 개발·학습 콘텐츠다. 한국어로 '학습 카드'를 만들어라.
원문에 있는 내용만 사용하고, 없는 사실·코드는 절대 지어내지 마라.
아래 형식을 그대로 지켜라. 마크다운 헤딩('#')은 절대 쓰지 말고, 굵은 라벨과 불릿만 써라:

**핵심 개념**
- (배워야 할 개념·원리 3~5개, 각 한 줄)

**코드·명령**
(원문에 코드/명령/CLI가 있을 때만. ```로 감싼 코드펜스로. 없으면 이 섹션 전체 생략)

**실습 포인트**
- (따라 할 수 있는 단계나 팁 2~4개)

**한 줄 정리**
(한 문장)

그 다음 마지막 줄에 '카테고리: '로 시작해 아래 목록 중 1개(최대 2개)만 쉼표로 적어라. 없으면 '개발·학습'.
카테고리 목록: {categories}

제목: {title}
출처: {source}
내용:
{body}
```

파싱은 기존 `summarize_and_classify`와 동일: `카테고리:` 줄만 분리하고 나머지가 요약.

## 변경 지점

| 파일 | 변경 |
|---|---|
| `sources.yaml` | 학습형 출처 추가 + `learning: true` |
| `collector/config.py` | `Source.learning: bool` 로딩 (기본 False) |
| `collector/models.py` | `Item.learning: bool` (기본 False) |
| `collector/pipeline.py` | 소스 `learning` → Item 전파 |
| `collector/summarize.py` | `item.learning`이면 `LEARNING_PROMPT` 분기 |
| `categories.yaml` | "개발·학습" 추가 |
| `web/app.py` | `<pre>/<code>` 다크 CSS, 학습 카드 뱃지(선택) |
| `tests/` | 플래그 전파·learning 분기·카테고리 |

## 스코프 밖 (YAGNI)

- 기존 소스의 항목별 학습 자동 판별 (소스 단위 플래그로 시작)
- 퀴즈/플래시카드
- RAG 인터랙티브 튜터 (C안 — 백로그)

## 리스크

- 자막·설명이 부실한 영상은 코드 스니펫 품질↓ → "코드·명령"은 **원문 근거 있을 때만**, 지어내기 금지.
- 학습 프롬프트도 무료 쿼터 사용 → 기존 키 로테이션 그대로.
