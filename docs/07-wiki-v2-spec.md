# 07. 위키 v2 요구사항 정의서 — 내용 강화 + 관리

> PM 단계: 요구사항 정의 (위키 v2). 작성 2026-06-29.
> v1.1(고정 7카테고리 + 목차 + 통합호출)은 가동 중. 이건 그 위에 **실제 내용을 보이게(B) + 운영 관리**를 얹는다.
> **2단계로 빌드:** v2.0(내용) 먼저 → v2.1(관리).

## 1. 목적 & 범위
- **목적:** 토픽 페이지에서 **클릭 없이 실제 내용(알맹이)**이 보이고, 위키를 **오래 운영·수정**할 수 있게 한다.
- **범위(포함):** 자세한 요약, 페이지에 요약 표시, 카테고리 외부설정, 내 메모 보존, 페이지 길이 관리.
- **범위(제외):** 다이어그램/ERD(v3), deep-research 검증(별도), 웹 공개(별도).

## v2.0 — 내용 강화 (먼저)

### FR (v2.0)
| ID | 요구사항 |
|---|---|
| FR-1 | 요약을 3줄이 아닌 **핵심 포인트 5~7개(불릿)**로 더 자세히 생성한다 (요약+분류 통합 호출 유지, 호출 수 불변) |
| FR-2 | 토픽 페이지의 "관련 소식"에서 각 항목 아래 **요약 내용을 펼쳐서 표시**한다 (제목·출처·날짜·원본링크 + 요약 불릿) |
| FR-3 | 기존에 쌓인 항목(3줄 요약)은 그대로 표시, 새 항목부터 자세한 요약 적용 (재요약으로 쿼터 낭비 안 함) |

### v2.0 페이지 형식 (관련 소식 부분)
```markdown
## 관련 소식
### [EP 100. Claude Mythos, Fable 5](링크)
노정석 · 2026-06-27
- Claude Mythos·Fable 5 공개로 경쟁축이 성능→안전·데이터로 이동
- 추론 인프라·토큰 경제학이 비용의 핵심 변수
- 한국 독자 모델 기회 ... (5~7개)
```

### v2.0 변경 파일
- `collector/summarize.py` — `summarize_and_classify`/`summarize_item` 프롬프트를 "핵심 포인트 5~7개 불릿"으로 강화.
- `collector/topics.py` — `render_page`의 관련 소식 렌더를 제목만 → **제목 + 출처/날짜 + 요약 불릿**으로. (요약은 이미 `topics.json` 항목에 저장돼 있음)

## v2.1 — 운영 관리 (다음)

### FR (v2.1)
| ID | 요구사항 |
|---|---|
| FR-4 | **카테고리를 `categories.yaml` 파일**에서 읽는다 (없으면 코드 기본값). 코드 수정 없이 카테고리 추가/이름변경 가능 |
| FR-5 | 각 토픽 페이지에 **`## ✍️ 내 메모`** 구역. 자동 재생성 시 **이 구역의 내 편집은 보존**(읽어서 다시 삽입), 나머지(개요·관련소식)만 갱신 |
| FR-6 | "관련 소식"은 **최근 N개(기본 30)만** 페이지에 표시; 초과분은 **월별 아카이브** `notes/topics/archive/<주제>-<YYYY-MM>.md`로 이동하고 본문에 링크 |

### v2.1 설계 포인트
- **categories.yaml:** `classify.CATEGORIES`를 `load_categories()`로 대체 (파일 있으면 거기서, 없으면 상수). 출처는 기존 `sources.yaml` 그대로.
- **메모 보존:** `render_page`가 페이지 하단에 `<!-- 내메모-시작 -->` ~ `<!-- 내메모-끝 -->` 마커 구역을 둠. `write_pages`는 기존 파일에서 그 구역을 읽어 새로 생성하는 페이지에 그대로 끼워넣음. (사용자는 그 구역 안에만 편집)
- **아카이브:** `render_page`는 최신 30개만 출력; `write_pages`(또는 topics)가 초과분을 월별 아카이브 파일로 쓰고 본문에 "이전 기록 N건 → [[archive/...]]" 링크.

## 2. 비기능 요구사항
- **비용:** 자세한 요약은 호출 수 불변(토큰만 증가) → 무료 한도 영향 최소. 카테고리/메모/아카이브는 LLM 호출 없음.
- **안정성:** categories.yaml 없거나 깨져도 기본값으로 동작. 메모 구역 없으면 새로 만듦. 아카이브 실패해도 본문은 정상.
- **정직성:** 요약은 원문 근거, 출처 링크 유지(추적성).

## 3. 성공 기준
- **v2.0:** 토픽 페이지를 열면 각 소식의 **핵심 내용 5~7개가 클릭 없이** 보인다.
- **v2.1:** `categories.yaml` 수정으로 카테고리가 바뀐다 / `## ✍️ 내 메모`에 적은 내용이 다음 자동 실행 후에도 **남아있다** / 관련 소식이 30개 넘으면 아카이브로 빠지고 본문은 깔끔하다.
