# 02. 요구사항 정의서 — 1단계 ✅ 확정

> PM 단계: **요구사항 정의 (확정)**. 결정 사항 모두 확정됨 (2026-06-27).
> 1단계만 다룹니다. (2~4단계는 나중에 별도 요구사항 작성)

## 1. 목적 & 범위 (1단계)

- **목적:** 매일 새 AI 정보를 자동 수집·요약해서 **아침에 요약본을 받아본다.**
- **범위 (포함):** 유튜브 + 뉴스레터 수집, 새 글만 감지, 한국어 요약, 마크다운 저장, 아침 요약 전달.
- **범위 (제외, 나중):** Q&A, 위키, Threads/X, 피드백 학습.

## 2. 기능 요구사항 (FR)

| ID | 요구사항 | 비고 |
|---|---|---|
| FR-1 | 지정한 유튜브 채널들의 **새 영상**을 매일 감지한다 | 채널 RSS 사용 |
| FR-2 | 새 영상의 **자막을 가져와 한국어로 요약**한다 | `watch` 스킬 |
| FR-3 | 지정한 뉴스레터의 **새 글**을 가져와 요약한다 | RSS/이메일 |
| FR-4 | "이미 처리한 항목"을 기록해 **중복 수집을 막는다** | 상태추적 |
| FR-5 | 요약 결과를 **날짜별 마크다운 파일**로 저장한다 | 저장 경로 = 아래 |
| FR-6 | 매일 정해진 시각에 **요약본을 나에게 전달**한다 | 전달 방법 = 아래 |

## 3. 확정된 결정들 ✅

### 3-1. 수집 출처 (1단계) ✅ 확정
- **유튜브:** 노정석(chester_roh), 윤자동(yunjadong), 조코딩(jocoding), TTimesTV, 안될공학(unrealtech)
- **뉴스레터:** SaaStr, Lenny's Newsletter, a16z, All Star SaaS(일본어)
- 출처 추가/삭제는 설정 한 곳 수정으로. (구현 시 채널 ID/RSS URL 확정)

### 3-2. 아침 요약 전달 방법 ✅ 확정
- **(A) 매일 마크다운 1장으로 `notes/daily/`에 저장** → 옵시디언/대시보드에서 읽기. 추가 계정 불필요.
- (텔레그램 푸시·이메일은 나중에 원하면 추가)

### 3-3. 저장 경로 (옵시디언) ✅ 확정
- **저장 위치:** `C:\ProjectS\ai-knowledge-assistant\notes\` (날짜별: `notes/daily/`)
- **Obsidian 볼트:** 프로젝트 폴더 `C:\ProjectS\ai-knowledge-assistant`를 볼트로 연다.
- **버전관리:** 이 폴더 전체를 git 저장소로 만들고 **GitHub(storm-credit)에 푸시**. 비밀키는 `.gitignore`로 제외.

### 3-4. 실행 시각 ✅ 확정
- **매일 새벽 5시** 자동 실행 (Hermes Cron).

### 3-5. 요약에 쓸 LLM 모델 ✅ 확정
- **1단계: `gemini-2.5-flash` (무료)** 로 시작 — 동작 확인됨. 빨리 굴려보고 품질 불만 시 나중에 교체(개인 OpenRouter 종량제 / 로컬 Ollama 등).

## 4. 비기능 요구사항

- **비용:** 무료 또는 소액. 회사 계정·구독 사용 금지.
- **안정성:** 한 출처 수집 실패해도 나머지는 계속 (개별 실패 격리).
- **로컬:** Hermes Docker 안에서 동작, 데이터는 내 PC에 저장.

## 5. 성공 기준 (1단계 "완료"의 정의)

- 아침에 일어나면, 전날~당일 새로 올라온 유튜브/뉴스레터가 **한국어 요약본 1장**으로 정리돼 있다.
- 같은 항목이 **중복으로 다시 나오지 않는다.**
- 출처를 추가/삭제하는 게 **설정 한 곳 수정**으로 된다.
