# 01. 컨셉 & 아키텍처

> PM 단계: **컨셉/아키텍처 (확정)**. 작성일 2026-06-26.

## 1. 컨셉

- **무엇:** AI 관련 정보를 매일 자동 수집 → AI가 요약·태깅 → 저장 → 아침 요약·Q&A·위키로 활용하는 **개인용 지식 비서**.
- **누구를 위해:** 나 (AI 시대에 안 뒤처지려고 정보를 챙겨보는 사람).
- **엔진:** Hermes Agent (로컬 Docker). 이게 "하네스" — 이미 결정됨.

## 2. 아키텍처 (데이터 흐름)

정보가 출처(위) → 나(아래)로 흐른다.

```
① 출처            유튜브 · 뉴스레터 (쉬움) / 웹 (보통) / Threads·X (어려움, 나중)
      ↓
🔖 상태추적        "이미 본 것" 기록 → 새 글만 가져오기 (중복 방지, 필수)
      ↓
② 수집            Hermes Cron 매일 새벽 자동 실행 (RSS / 자막 / 메일)
      ↓
③ 처리            LLM이 한국어 요약 + 주제 태그
      ↓
④ 저장            날짜별 마크다운 (옵시디언 볼트) (+ 나중에 벡터DB 검토)
      ↓
⑤ 활용            📩 아침 요약 푸시 / 💬 Q&A(RAG) / 🕸 위키·지식그래프
      ↑
⭐ 피드백 루프      "중요/별로" 표시 → 다음 요약 취향 맞춤 (선택, 포함하기로 함)
```

## 3. 부품 ↔ 도구 매핑

| 부품 | 담당 |
|---|---|
| ② 수집 자동실행 | Hermes Cron |
| 유튜브 자막·요약 | `watch` 스킬 |
| ④ 저장/기억·회상 | `agentmemory` + 마크다운 파일 |
| ⑤ 위키/지식그래프 | `understand-anything` 스킬 |
| 웹·기사 수집 | `deep-research` / 웹 도구 |
| 전체 설계·구현 방법론 | `superpowers` |

## 4. 단계별 로드맵 (한 번에 다 안 만든다)

| 단계 | 범위 | 산출 |
|---|---|---|
| **1단계** | 유튜브+뉴스레터 → 상태추적 → 수집 → 요약 → 저장 → **아침 요약** | 매일 쓸모 있는 비서 |
| 2단계 | 쌓인 지식에 **Q&A(RAG)** | 질문하면 답하는 비서 |
| 3단계 | **위키/지식그래프** 탐색 | 옵시디언 그래프 |
| 4단계 | **어려운 출처(Threads/X)** + 피드백 학습 고도화 | 완성형 |

## 5. 기술 결정 원칙 (WHAT vs HOW)

- 이 문서까지는 **WHAT** (무엇을). 기술 선택(HOW)은 3~4단계에서.
- **하네스:** Hermes ✅ (결정됨)
- **LangChain:** 아마 **안 씀** — Hermes가 이미 에이전트 루프·도구·메모리를 제공. 중복.
- **벡터DB:** 1단계 불필요. 2단계에서 양 보고 결정 (안 쓸 수도).
- **프롬프트/컨텍스트/에이전틱 엔지니어링:** 4단계 구현하며 튜닝.
