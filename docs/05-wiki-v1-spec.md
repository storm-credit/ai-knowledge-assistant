# 05. 주제별 자동정리 위키 — v1 요구사항 정의서

> PM 단계: 요구사항 정의 (3단계 위키, v1). 작성 2026-06-27.
> 1단계(수집·요약·일별저장) 위에, 수집된 항목을 **주제별로 자동 분류·정리**해 나무위키식 페이지를 만든다.

## 1. 목적 & 범위

- **목적:** 매일 쌓이는 요약을 **주제별 페이지**(`AI에이전트.md`, `Claude.md`…)로 자동 정리해서, 날짜가 아니라 **주제로 찾아보는 위키**를 만든다.
- **범위 (v1 포함):** 주제 자동 분류, 주제 페이지 생성/갱신(하이브리드), 출처 링크, "N개 출처" 신뢰 신호, 주제 간 링크(`[[ ]]`).
- **범위 (제외 → v2/v3 백로그):** Mermaid 다이어그램/ERD(v2), deep-research 팩트체크(v3), 웹 공개(별도).

## 2. 기능 요구사항 (FR)

| ID | 요구사항 |
|---|---|
| FR-1 | 아직 분류 안 된 수집 항목을 찾아, **LLM이 1~3개 주제로 분류**한다 |
| FR-2 | 주제명은 **기존 주제 목록에 맞춰 정규화**한다 (예: "에이전트"→기존 "AI 에이전트"로 흡수, 파편화 방지) |
| FR-3 | 각 주제별 페이지를 `notes/topics/<주제>.md` 로 생성/갱신한다 (Obsidian 볼트 내) |
| FR-4 | **하이브리드 갱신**: 새 항목은 즉시 "관련 소식"에 추가; "개요"는 새 항목 ≥5개 누적 시(또는 `--resynth`) LLM이 재합성 |
| FR-5 | 모든 항목에 **출처 링크** 포함; 페이지 상단에 **"N개 출처에서 언급"** 표시 |
| FR-6 | LLM이 제안한 **관련 주제를 `[[링크]]`**로 연결 (그래프 형성) |
| FR-7 | 이미 처리한 항목은 다시 분류하지 않는다 (상태추적) |
| FR-8 | LLM 호출은 **flash-lite + 3키 로테이션** 재사용 |

## 3. 확정된 결정

- **갱신 방식:** D. 하이브리드 (새 항목 즉시 추가 + 개요 주기적 재합성).
- **검증:** 출처 추적(링크) + 다출처 신호("N개 출처"). **자동 팩트체크 없음**(v3 옵션).
- **저장 위치:** `notes/topics/*.md` (볼트 내 → Obsidian `[[링크]]`·그래프로 탐색).
- **모델:** gemini-2.5-flash-lite (3키 로테이션).
- **이력:** git이 모든 변경 버전관리(공짜).

## 4. 주제 페이지 형식 (예시)

```markdown
# AI 에이전트
> 📌 3개 출처에서 언급 · 최근 갱신 2026-06-27

## 개요
AI가 실행을 담당하고 인간은 '의도'를 제공하는 패러다임으로 이동 중...
(LLM이 관련 항목들을 합성)

## 관련 소식
- 2026-06-27 · 노정석 EP98: AI 실행 시대, 인간에겐 의도만 — [출처](링크)
- 2026-06-27 · 조코딩: 코덱스 활용법 — [출처](링크)

## 관련 주제
[[Claude]] · [[바이브코딩]] · [[SaaS]]
```

## 5. 아키텍처

```
수집·요약(1단계) → notes/daily/*.md
        ↓  (새 항목)
[분류]  collector/classify.py
   LLM이 각 항목 → 1~3개 정규화된 주제 (기존 주제 목록 참고)
        ↓
[위키 갱신]  collector/wiki.py
   주제별로: 관련소식 추가(즉시) + 개요 재합성(조건부) + 출처 카운트 + [[링크]]
        ↓  → notes/topics/<주제>.md
[실행]  python -m collector wiki   (+ 매일 run 뒤 자동 호출)
[상태]  state/classified.json (분류 완료 항목 ID)  ·  state/topics.json (주제 정규화 인덱스)
```
- **새 파일:** `collector/classify.py`(분류), `collector/wiki.py`(페이지 생성·갱신). 기존 summarize(키 로테이션)·마크다운 파싱 재사용.
- **데이터모델:** Topic(name, aliases, source_count, items[], overview, related[]).

## 6. 비기능 요구사항
- **비용:** 무료(기존 키). 분류는 항목당 1회, 개요 재합성은 조건부(비용 절약).
- **안정성:** 개별 항목 분류 실패해도 나머지 계속; 실패 항목은 미분류로 남겨 다음에 재시도.
- **정직성:** 개요는 관련 항목 **원문에 근거**, 없는 사실 생성 금지. 모든 주장에 출처 추적 가능.

## 7. 성공 기준 (v1 완료)
- `python -m collector wiki` 실행 시, 그날 수집 항목이 **주제별 페이지로 정리**되어 `notes/topics/`에 생성·갱신된다.
- 같은 주제에 새 글이 오면 **중복 없이 누적**되고 출처 수가 는다.
- 옵시디언에서 주제 페이지를 `[[링크]]`로 타고 다니며 그래프로 볼 수 있다.
- 매일 5시 자동 실행에 위키 갱신이 포함된다.
