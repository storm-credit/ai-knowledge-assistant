# 프롬프트 엔지니어링

> 생성일 eval · 적용형 학습 노트

## 이게 뭔가
프롬프트 엔지니어링은 대규모 언어 모델(LLM)이나 생성형 AI와 같은 인공지능 모델에게 원하는 결과물을 얻기 위해 효과적인 질문(프롬프트)을 설계하고 다듬는 기술입니다. 최근 AI 기술의 발전으로 인해 프롬프트 엔지니어링은 AI를 실질적으로 활용하는 핵심 역량으로 주목받고 있습니다. 단순히 질문하는 것을 넘어, AI의 잠재력을 최대한 이끌어내고 편향되거나 부정확한 응답을 줄이는 데 중요한 역할을 합니다.

## 어떻게 배우나
*   **기초 개념 이해:** LLM이 어떻게 작동하는지, 프롬프트의 중요성이 무엇인지 학습합니다. AI 모델의 작동 방식을 알아야 왜 특정 프롬프트가 더 효과적인지 이해할 수 있습니다.
*   **다양한 프롬프트 실험:** 간단한 질문부터 시작하여 복잡한 지시사항, 제약 조건 등을 포함한 다양한 프롬프트를 직접 입력하며 AI의 응답 변화를 관찰합니다. 실습을 통해 이론을 체득하는 것이 중요합니다.
*   **결과 분석 및 개선:** AI의 응답을 비판적으로 검토하고, 기대치에 미치지 못했을 때 프롬프트를 어떻게 수정해야 하는지 학습합니다. 원하는 결과에 도달할 때까지 프롬프트를 반복적으로 개선하는 연습을 합니다.
*   **고급 기법 학습:** Few-shot learning, Chain-of-thought, Role-playing 등 고급 프롬프트 기법을 익히고 적용해 봅니다. 복잡한 문제를 해결하거나 창의적인 결과물을 얻는 데 도움이 됩니다.

## 내 적용법
*   **파이썬 기반 자동화 도구 개선:** 기존에 파이썬으로 만든 개인 자동화 도구에 Claude Code와 같은 AI 모델을 통합할 때, 더 정확하고 효율적인 코드 생성을 위해 프롬프트 엔지니어링을 적용합니다. 원하는 기능, 제약 조건, 코드 스타일 등을 명확하게 지정하는 프롬프트를 설계합니다. (제안)
*   **PM 방식 워크플로우에 AI 활용:** PM(Project Manager) 방식으로 일할 때, 스펙 정의 단계에서 AI에게 요구사항을 명확히 전달하고, 위임 단계에서 AI가 특정 작업을 수행하도록 구체적인 지침을 제공하며, 검증 단계에서 AI의 결과물을 효과적으로 평가하는 데 프롬프트 엔지니어링을 활용합니다. (제안)
*   **복잡한 작업 자동화를 위한 프롬프트 설계:** 여러 단계를 거쳐야 하는 복잡한 자동화 작업(예: 데이터 분석, 보고서 초안 작성)을 AI에게 맡길 때, 단계별로 명확한 지시와 예시를 제공하는 연쇄적인 프롬프트(Chain-of-thought)를 설계하여 정확도를 높입니다. (제안)
