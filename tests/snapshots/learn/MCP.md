# MCP

> 생성일 eval · 적용형 학습 노트

## 이게 뭔가
MCP는 'Model-Centric Programming'의 약자로, AI 모델 자체의 개발과 개선에 초점을 맞추는 프로그래밍 패러다임을 의미합니다. 기존에는 데이터 중심 프로그래밍(Data-Centric Programming)이 데이터 전처리나 라벨링 개선에 집중했다면, MCP는 모델의 구조, 학습 방식, 성능 최적화 등 모델 자체의 성능 향상에 더 큰 비중을 둡니다. 최근 AI 모델의 복잡성과 성능 요구치가 높아지면서, 모델 자체를 효율적으로 다루고 개선하는 MCP의 중요성이 부각되고 있습니다. (불확실)

## 어떻게 배우나
*   **기본 AI 모델 이해:** 지도 학습, 비지도 학습, 강화 학습 등 기본적인 AI 모델의 원리와 작동 방식을 학습합니다. 모델의 핵심 동작 방식을 이해하는 것이 MCP의 시작점입니다.
*   **모델 아키텍처 탐구:** CNN, RNN, Transformer 등 다양한 신경망 아키텍처의 특징과 장단점을 파악합니다. 문제에 맞는 적절한 아키텍처 선택 능력을 기르는 것이 중요합니다.
*   **학습 알고리즘 및 최적화 기법 익히기:** 경사 하강법, Adam 등 최적화 알고리즘과 정규화, 드롭아웃 등의 기법을 학습합니다. 모델의 성능을 효과적으로 향상시키는 방법을 배웁니다.
*   **모델 평가 지표 이해 및 활용:** 정확도, 정밀도, 재현율, F1 점수 등 다양한 평가 지표를 이해하고, 실제 모델 성능을 객관적으로 측정하고 개선 방향을 설정하는 연습을 합니다.

## 내 적용법
*   개인 자동화 도구를 만들 때, Claude Code와 같은 AI 모델의 내부 구조나 작동 방식에 대한 이해를 바탕으로 프롬프트 엔지니어링을 넘어 모델 자체의 파라미터를 조정하거나 특정 기능을 활용하는 방식을 시도해 볼 수 있습니다. (제안)
*   PM 방식으로 일할 때, 스펙 작성 시 자동화 도구에 필요한 AI 모델의 성능 요구사항을 구체적으로 정의하고, 이후 검증 단계에서 모델의 약점을 파악하여 개선점을 도출하는 데 MCP 지식을 활용합니다. (제안)
*   만약 개인 자동화 도구에 커스터마이징이 필요한 부분이 있다면, 기존 모델의 아키텍처를 분석하거나 특정 학습 기법을 적용하여 도구의 성능을 더욱 향상시키는 시도를 할 수 있습니다. (제안)
